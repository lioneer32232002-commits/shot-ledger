## 驗收方法論（Fable 必跑，血淚換來的）

1. **分享卡垂直帶掃描**：2/4 球種 × 紙感/照片 4 張，逐 y 掃墨水帶，
   相鄰帶間 ≥16px（M4.3 前漏檢造成實機重疊，使用者抓到）。
   左右兩欄佈局要**分欄掃**，全寬掃會誤報。
2. 對齊掃描：跨行顆數右緣、% 右緣像素級切齊。
3. 本機 QA 快取三層陷阱：dev server 掛掉時 **SW 離線快取會餵舊檔**
   （network-first 斷網 fallback，看起來像沒改到）；瀏覽器對 js 模組有
   **HTTP 啟發式快取**（python http.server 只給 Last-Modified）。
   驗收前固定：unregister SW → caches 全清 → 逐檔 fetch {cache:'reload'}
   → reload。
4. `hidden` 屬性會被任何 display class 蓋掉——app.css 已加
   `[hidden]{display:none!important}` 根治，新元件別再踩。
