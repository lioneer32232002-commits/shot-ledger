# SPEC_M12：徽章改成「刊號數字盤」

> 使用者回饋（2026-07-21）：17 顆徽章只用 4 種線條圖示，同家族好幾顆長一樣，
> 在**沒有文字說明的生涯分享卡**上看起來像壞掉。第一版比稿（每顆各畫一個具體
> 物件：球袋、籃框網、記分板、看臺…）被否決——「太抽象、不好看、沒有驚艷感」，
> 24px 線條圖示本來就畫不出這些東西。
>
> 定案方向：**讓門檻數字當主角**（比稿 A 案，含使用者拍板的兩點修正）。
> 這同時解決根本問題——別人在分享卡上直接讀得到「30 天」「10K 球」。

---

## 0. 設計規格

每顆徽章＝**數字盤**，由外而內：

```
   ┌ 進度環（既有，未獲得時顯示進度）
   │  ┌ 圓盤底（accent 淡色填底＋accent 描邊）
   │  │   30天      ← 中央：門檻數字（800 字重）＋單位小字
   │  │    🔥       ← 下方：家族小標記（沿用現有 ICON_PATH 四個圖示）
```

### 0.1 四個家族的數字與單位

| 家族 | 徽章 | 數字盤內容 | 單位 |
|---|---|---|---|
| 出席 `streak` | 3／7／14／30／60 | `3` `7` `14` `30` `60` | `天` |
| 投量 `volume` | 1000…50000 | `1K` `2.5K` `5K` `10K` `25K` `50K` | 無（K 已表意） |
| 摘星 `stars` | 10／25／全滿 | `10` `25` `45` | `★` ←**使用者拍板加上** |
| 階梯 `ladder` | 3／7／全破 | `3` `7` `15` | `關` |

- **動態門檻不得寫死**：`stars_full` ＝ `ladder.length × 3`、`ladder_complete` ＝
  `ladder.length`（現為 45／15，加關卡就自動變）。這兩顆的數字必須由
  `ladderMenus()` 算出來——寫死會在下次加關時默默錯掉。
- 投量的 K 換算：`1000→1K`、`2500→2.5K`、`10000→10K`（整數不補 `.0`）。
- 家族小標記沿用 `ICON_PATH` 的 `flame`／`ball`／`star`／`trophy`，**不重畫**。
- 出席的 `3`／階梯的 `3`（`7` 同理）靠單位字區分，不能只靠家族標記——比稿實測
  44px 下家族標記只有約 9px，分不出火焰與獎盃。

### 0.2 排版鐵律

- **一個尺寸只有一個基準字級**：先量最寬的字串（`2.5K`）決定該尺寸的字級，
  同尺寸下所有徽章共用，不因位數不同而忽大忽小。
- **視覺置中**：數字沒有下伸部，幾何置中會偏低。canvas 端用既有
  `measureAscDesc()` 的墨水高度做光學置中；DOM 端也用同一套思路（不要用魔術常數）。
- 數字與單位以基線對齊，單位字級約為數字的 45%。

## 1. `js/badges.js`

1. `BADGE_DEFS` 每顆補上家族資訊即可（`kind` 已存在），**新增純函式**：
   ```js
   /** 徽章的數字盤內容：{ num, unit, icon }。動態門檻（stars_full／ladder_complete）
    *  依現行階梯長度算，不得寫死。 */
   export function badgeNumeral(def)   // 或 badgeNumeral(id)，實作者自行取捨
   ```
2. `medalHtml()` 改畫數字盤：進度環保留不動，圖示區換成
   `數字＋單位`＋下方家族標記。徽章牆上的名稱與進度文字**維持原樣**（牆上本來就有說明，
   數字盤只取代圖示）。
3. `badgeStripHtml()` 的 44px 迷你盤同步改成數字盤。
4. `earnedBadgeList()` 回傳值補上 `num` / `unit`（生涯分享卡要用）。

## 2. `js/sharecard.js`

1. `drawBadgeMedal()` 改畫數字盤（數字＋單位＋家族小標記），沿用既有圓盤語彙
   （accent 8% 填底＋accent 2px 描邊）與 `drawBadgeIconShape()` 的 SVG 解析法。
2. **排列改三列（使用者拍板）**：一列 7 顆、直徑 112、間距 24（7×112＋6×24＝928 ＝
   卡片內容寬）。
   - ≤7 顆：一列
   - 8–14 顆：兩列
   - **15 顆以上：三列（7＋7＋3…）**，最多顯示 21 顆
   - 目前徽章總數 17，三列放得下 → **「＋N」收尾格在現行資料下不會出現**；
     但保留 >21 顆時的 `＋N` 保底（未來徽章再擴充才會觸發）。
   - 列距 24。
3. `computeLifetimeLayout()` 的高度計算要把列數變化算進去（三列比兩列高 136px）；
   空間不足時仍照既有規則先縮圓盤（112 → 最小 76），**絕不重疊**。

## 3. `css/app.css`

新增數字盤的 class（`.badge-medal__num`／`__unit`／`__mark` 或同等命名），
三種尺寸（成就條 44／徽章牆 68／分享卡由 canvas 畫）。顏色一律走 tokens 變數。

## 4. `tools/check_ladder.mjs`

新增一項檢查：`js/badges.js` 內不得出現寫死的 `45` / `15` 當作 `stars_full` /
`ladder_complete` 的數字盤門檻（正則掃 `badgeNumeral` 附近的字面數字），
抓到就 FAIL 並提示「動態門檻要用 ladderMenus() 算」。
