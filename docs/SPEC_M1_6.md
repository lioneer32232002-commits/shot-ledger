# SPEC M1.6 — 排版紀律＋極速啟動＋自訂等級對照＋累計數（2026-07-12）

前提：M1.5 已完成驗收。本規格為小增量，沿用全部既有慣例。

## 1. 排版紀律（全 App 通掃）

- **禁止孤字/孤符號換行**：任何「數字＋單位」「數值＋符號」的資訊單元
  （`59%`、`53/90 投中`、`約 30 分`、`7/12（週日）`、`第 3 / 9 輪`）
  必須整體不可拆行——包在 `white-space: nowrap` 的 span（建 utility class
  `.nowrap`）。同一筆資訊盡量同一行呈現。
- 多行標題與說明文字加 `text-wrap: balance`（標題）/ `pretty`（內文），
  避免第二行只剩 1–2 個字。
- 驗收：375px 與 390px 寬逐頁檢查（首頁、變體選擇、session、統計、紀錄、
  設定），不得出現第二行只有孤字或孤符號的情況；容不下就縮字級或改寫文案，
  不是放任換行。

## 2. 極速啟動（點擊數硬上限）

- **規則：從開啟 App 到「可以開始投球」≤ 3 次點擊**（自訂目標流程除外）。
- 挑戰：首頁 hero「開始挑戰」→ 變體 sheet 點「簡易/完整」→ 即開始（2 點）。
  變體 sheet 不加多餘的「確認」步驟。
- 自由練習：點卡片 → 即開始（1 點；預設「不指定位置＋上次球種」，
  選點是加分動作不是前置關卡）。
- 快速重開：首頁 hero 下方加一行「▶ 再來一次：{上次菜單＋變體}」（1 點直開）。
  沒有歷史時不顯示。
- 驗收：實際數挑戰、自由、快速重開三條路的點擊數，逐條回報。

## 3. 自訂（自由/綜合）練完的等級對照

- stats.js 新純函式 `equivalentTier(session, menus)`：
  由高關往低關掃挑戰菜單的 passRule——該節在 rule 內**每個球種 attempts ≥ 20**
  且 pct 全數達標 → 回傳該關 {tier, menuId, name}；都不滿足回 null。
- 非挑戰節（free / world）的結束頁與歷史詳情加「等級對照」卡：
  - 命中：「本節表現相當於 第 {tier} 關・{名稱} 的過關水準」
    ＋固定小字「僅供對照，不解鎖關卡——挑戰請照階梯一關一關過」。
  - 樣本不足或未達第 1 關 → 整卡隱藏（不要顯示負面訊息）。
- **絕不寫入 progress.unlocked**——對照歸對照，解鎖歸解鎖。
- 測試補：equivalentTier（樣本不足、跨球種 rule、剛好壓線、全不符）。

## 4. 累計總數（生涯數字要隨時看得到）

- 首頁今日小結區加第二行：「累計 {N} 投 / {M} 中（{pct}%）」
  （千分位逗號、`.nowrap`）。全生涯所有節（含自由）都計入。
- stats.js `lifetimeTotals(sessions)` → {att, mk}（含測試）。
- 設定頁資料狀態區同步顯示同一數字。

## 5. 驗收清單

1. 六個主要畫面 375/390px 無孤字孤符號換行；資訊單元不拆行。
2. 挑戰 2 點開始、自由 1 點開始、快速重開 1 點開始，實測回報。
3. 自由節打出高命中 → 等級對照卡正確顯示對應關卡且 progress 不變；
   樣本不足時卡片隱藏。
4. 首頁與設定頁累計數一致且與手算相符。
5. `node tools/test_stats.mjs` 全過（含新測試）；console 零錯誤。
