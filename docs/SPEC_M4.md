# SPEC M4 — 實測回饋修正（2026-07-13 使用者實跑一輪完整版後的 8 點回饋）

> 分工：Fable 寫本 spec → Sonnet 實作 → Fable 瀏覽器實測驗收。
> 鐵律不變：vanilla JS ES modules、無 build step、不動 menus.js 的數據與出處、回覆與 UI 文案繁體中文。
> 「簡單有質感」優先於功能堆疊——每一項都以質感為驗收標準。

## 0. 背景：本次實測發現的核心 bug（№4 的根因）

使用者跑完整版最後一輪時，準備按進球數，卻發現「已經統計了」。
根因在 [session.js](../js/session.js) 的 `completeRound()`：菜單模式下記錄完**最後一輪**後
直接呼叫 `finishSession()`，零確認、零緩衝。使用者其實是不小心碰到某個
進球數按鈕（快速模式整片都是大按鈕），那一下就直接結算了。

App **沒有時間上限**——計時器只是顯示經過時間；「時間到」不存在。
時間只有下限（誠實機制：完整版 ≥20 分才列入解鎖評估），所以這次被提早
結算 → 時長不足 → 挑戰不通過。修好 №4 後此情境不會再發生。

## 1. 選項類按鈕質感升級（去掉「AI 風格長條」）

現況：變體選擇（簡易版/完整版）、快速/逐球切換等都是全寬長方形按鈕，
呆板且同質。質感基準是首頁的 `SHOT` 圓形 CTA（`.hero-card__cta-circle`：
accent 實色＋外圈 1.5px 半透明 ring＋按壓 scale）。

改造對象與方向（只動 CSS＋必要的 HTML 結構，不改行為）：

- **變體選擇 sheet 的「簡易版／完整版」**（`.variant-option`）：
  改為兩張**並排方卡**（grid 兩欄），名稱大字置中、meta 小字在下，
  按壓時 accent 邊框＋淡 accent 底＋scale(0.97)。可加外圈 ring 呼應 SHOT 鈕。
- **快速／逐球切換**（`.input-mode-toggle`）：改成膠囊 segmented control——
  外框圓角 full、內部兩格、選中那格是 accent 實色圓膠囊（可用絕對定位滑塊
  或簡單背景切換），不再是兩顆分離的長鈕。
- **實投球數 stepper**（`.stepper-btn`）與**進球數鍵盤**（`.makes-btn`）：
  改成**圓形**（或近圓 squircle）數字鈕，比照球的意象；觸控目標維持 ≥44px。
- 檢查其他仍是「全寬長條」的選項類按鈕（chips 已是膠囊可不動；
  `.btn--primary/.btn--ghost` 等**動作**按鈕維持現狀，本項只針對**選項**類）。

驗收：手機寬度（390px）下無跑版、觸控目標不縮水、深色模式兩套都好看。

## 2. 正在投的點：淡淡發光

- 菜單模式從左側開始是**刻意設計**（動線由左往右），不改順序。
- `pick` 模式下目前選中的點（`.court-spot.is-selected .spot-dot`）加**柔和光暈**：
  CSS `filter: drop-shadow(0 0 6px var(--color-accent))` ＋緩慢呼吸動畫
  （opacity 或 shadow 強度 2–3s 循環，幅度小、不搶眼）。
- `prefers-reduced-motion: reduce` 時停用動畫、保留靜態光暈。
- 深色模式下光暈顏色仍用 accent，確認對比。

## 3. 輪次標示更醒目

- 現況「第 X / Y 輪」只在 header 小字，使用者實測時沒注意到。
  把輪次移進（或複製到）**輸入區塊**（`.round-input__head`）：
  「第 3 輪 / 12」用清楚的字級（不小於 text-sm、粗體），和點位名稱同區。
- 修改上一輪的功能其實已存在（「已完成（點一下可修改）」清單），
  但最後一輪被自動結算吃掉了（№4）。№4 修好後，所有輪次
  （含最後一輪）在結算前都可修改，本項即完成。
  順便把「已完成（點一下可修改）」的可點性做明顯一點
  （例如每列右側加「✎」或 chevron 提示）。

## 4. 移除最後一輪自動結算（本次最重要）

行為改成：

- `completeRound()` 記錄完最後一輪後**不再**呼叫 `finishSession()`，
  改設一個模組層級狀態（如 `menuComplete = true`）並 `renderActive()`。
- 完成狀態下的 `renderActive()`：輸入區換成**完成面板**——
  「12 輪全部完成 🎉 確認無誤後結算」＋一顆明顯的
  **「結束並結算」**primary 按鈕（呼叫 `finishSession()`）。
  已完成輪次清單照常顯示、照常可點修改（含最後一輪）。
  計時器照跑（時長是誠實機制的依據，結算才停）。
- `resumeSession()` 裡 `nextIdx >= seqList.length` 的分支同步改：
  不直接 `finishSession()`，改進入上述完成狀態。
- `startSession()` 時重置 `menuComplete = false`；`unmount()` 一併清理。
- 自由模式（無 seqList）行為不變（本來就要手按「結束練習」）。

驗收：跑一輪完整版到底，最後一輪按完數字**不會**跳統計；
可以點最後一輪修改；按「結束並結算」才進 summary。

## 5. 快速／逐球：一行說明

- 模式切換下方加一行小字（`text-xs`、muted 色），依當前模式顯示：
  - 快速：「只記這輪**進了幾球**」
  - 逐球：「記這輪**哪幾球**有進（點亮有進的球）」
- 切換模式時文字即時更新。一行就好，不做 tooltip 或彈窗。

## 6. 球場點位校正（底角三分回到線上）

[court.js](../js/court.js) 座標系：1m = 50px、籃框 (375, 79)、三分邊線 x=45 與 x=705、
三分弧半徑 337.5。已驗算：45° 三分與弧頂到籃框距離正好 337.5 ✓、
深三 ≈413px（8.26m）合理 ✓、中距各點皆在線內 ✓。**只有兩個底角三分偏離**：

- `3pt_lc`：cx **60 → 45**（點心落在左邊線上）
- `3pt_rc`：cx **690 → 705**

連帶要處理：**底角點的文字標籤**（`.spot-label` 目前 text-anchor middle，
cx=45 時「左底角三分」會超出 viewBox 左緣）。左底角改 anchor `start`、
右底角改 anchor `end`（x 各往內縮一點），或等效做法——驗收標準是
兩個標籤完整可見、不壓到線。分享卡的 `drawMiniCourt` 直接吃 SPOTS
座標，改完自動同步，無需另改（確認即可）。

## 7. 統計熱區點：在線上＋顯示命中率

- 位置隨 №6 自動修正。
- `heat` 模式的點（r=24）內容改成**兩行**：
  - 上行：**命中率**（如「70%」，粗體、稍大，tabular-nums）
  - 下行：投中數「7/10」（小字）
  白字壓在熱區色上，確認三種熱區色（cold/warm/hot）對比都夠；
  沒有出手的點維持現狀（無文字、idle 色）。
- 排版置中對齊、兩行間距緊湊不擁擠；手機寬度實看不糊。
- 分享卡 `drawMiniCourt` 的熱區點也比照：點內畫命中率%（canvas 版
  可只畫 %，空間比 SVG 小；字要清晰）。

## 8. 成績卡質感＋自訂照片背景

[sharecard.js](../js/sharecard.js) 現況是純色紙感卡。新增**照片模式**：

- 分享 sheet 加「**用自己的照片當背景**」按鈕（`<input type="file" accept="image/*">`，
  觸發選圖；另有「移除照片」可切回紙感版）。
- 選圖後重繪卡片：
  - 照片 **cover 裁切**鋪滿 1080×1350（置中裁切，短邊貼齊）。
  - 壓一層**漸層暗化**：整面 `rgba(0,0,0,0.35)`
    ＋上下加深的 linear-gradient（頂 0.55 → 中 0.25 → 底 0.65），
    確保品牌列與底部網址永遠可讀。
  - 文字全面改**亮色系**：主文字 #FAF9F7、muted 用半透明白
    （rgba(255,255,255,0.72)）、accent 維持 #E8590C（暗底上夠亮）。
  - 迷你球場線改半透明白；熱區點顏色不變（三色在暗底上都成立）。
- `drawCard(canvas, data, opts)` 加 `opts.photoImg`（HTMLImageElement|null），
  維持純函式；photo 只留在記憶體，**不進 localStorage**。
- 重繪後 dataURL/blob/File 全部重建，分享與下載拿到的是最新版。
- 無照片時輸出與現況一致（紙感亮色版不得回歸劣化）。

驗收：直向/橫向照片各試一張，文字永遠可讀；下載 PNG 打開檢查。
