## 0. 根因

- 熱區點內文字：`.spot-heat-pct` 15px、`.spot-heat-mkatt` 10px 是 SVG 座標系
  （viewBox 750 寬），手機上整個 SVG 只縮到約 343px 寬（比例 0.46），
  實際字高只剩約 7px / 4.6px——物理上不可能看清。
- 罰球線「兩個沒重疊好的點」：不是同點投兩輪（同點多輪本來就合併同一顆），
  是**兩個不同定點**幾乎同座標——`mid_top`（罰球線頂中距，375,300）與
  `ft`（罰球，375,310）只差 10px，兩顆 r=24 的點 96% 疊在一起，看起來像 bug。
- 分享卡：迷你球場只佔卡寬 70%、點 r=20、% 字 15px，都偏小。

## 1. 罰球點與罰球線頂中距分離（[court.js](../js/court.js) SPOTS）

- `ft`：cy **310 → 290**（正好壓在罰球線上：罰球線距籃框中心 4.225m ≈ 211px，
  79+211=290，物理正確）。
- `mid_top`：cy **300 → 350**（罰球線後一步的頂端中距，約 5.4m，
  與罰球點在圖上明確分開）。
- 更新檔頭／行內註解說明兩點距離依據。
- 驗算相鄰點間距：改完後任兩點中心距 ≥60px（供 §2 的 r=30 熱區點不相碰）。
  已預算過：ft(290)–mid_top(350)=60、mid_top(350)–3pt_top(416)=66、
  paint(145)–ft(290)=145、3pt_lw(136,317)–deep_l(83,370)≈75，皆過。
- 分享卡 `drawMiniCourt` 直接讀 SPOTS，自動同步（確認即可）。
- 順帶效益：pick 模式選點時罰球／中距頂不再互相誤觸。

## 2. 統計球場（heat 模式）易讀性重做

採「點內只放大字命中率＋點擊看完整資訊」：

- **有出手資料的點**：半徑 24 → **30**；點內**只放命中率**一行
  （`.spot-heat-pct` 15px → **26px**、字重 800；「100%」四字元時縮到 22px
  以免爆框）。移除點內第二行 mk/att（`.spot-heat-mkatt` 不再畫在點內）。
- **沒有資料的點**：半徑縮到 **8**、維持 idle 色——當背景參考，降低噪音。
- **點擊看詳細**：heat 模式的點（有資料者）可點（保留 r=22 透明 hit circle，
  縮小的無資料點不可點）。點擊後在球場 SVG 正下方顯示 HTML 資訊列
  `.court-info`（court.js 的 heat 模式自己在 container 內 svg 後面放一個 div，
  自我管理，不改呼叫端介面）：
  - 內容：「**罰球線頂中距**　14/20 ・ 70%」——真實 HTML 字級
    （text-sm 以上），數字 tabular-nums，命中率用該點熱區色強調。
  - 被點中的點加細白描邊（stroke）表示選中；點另一顆就切換。
  - 預設狀態（尚未點任何點）：資訊列顯示淡色提示「點場上的點看各點詳細」。
  - 資訊列高度固定（min-height），切換時版面不跳動。
- a11y：有資料的點 `role="button"`、`tabindex="0"`、Enter/空白可觸發，
  `aria-label` 帶完整資訊（點名、投中數、命中率）。
- 深色模式：資訊列用既有 surface/text tokens，兩套主題檢查。
- 此區塊會同時出現在「練球結束頁」與「紀錄分頁節詳情」（共用
  renderSessionSummary → renderCourt heat 模式），兩處自動生效，不用分別改。

## 3. 分享卡放大（[sharecard.js](../js/sharecard.js)）

目標：球場與點內 % 明顯變大，且**總高度不溢出** 1080×1350、底部網址保留。

- 迷你球場：卡寬 70% → **約 82–85%**（scale ≈ 1.2）。
- 熱區點：r 20 → **26**（SVG 座標，乘 scale 後約 33px）；點內 % 字
  15 → **18**（乘 scale 後約 22px）。
- 高度會多出約 100px，上方版面同步緊縮騰空間（方向建議，可微調）：
  - 球種列（typeRows）從一行一種改成**兩欄 grid**（最多 2×2），
    每列行高 50 → 44；
  - 主數字區塊 176 → 約 150（150px 大字不變，壓底下留白）；
  - 品牌列後 78 → 70、菜單行後 66 → 58 的間距微縮。
- 驗收標準：0–4 種球種都排得下（最多 2pt/3pt/deep3/ft 四行資料）、
  徽章列與底部網址不重疊、照片模式與紙感模式都檢查。
