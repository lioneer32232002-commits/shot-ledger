  區塊從 176 壓到 150，但「投中」小字 baseline 在 y+122，球種列首行離它
  只剩約 28px，36px 字直接撞上。上輪驗收只掃了欄位對齊與總高，沒掃
  「區塊間空白帶」，這次列為必檢。

- 快速模式數字鍵第二排「6 7 8 9 ＿ 10」：makesGridHtml 的 span 補位邏輯
  在圓鈕下變成尾端跳格，視覺突兀。

## 1. 數字圓鈕排列（[session.js](../js/session.js) makesGridHtml＋stepper）

- `makesGridHtml`（0–attempts）改成**列式排版**：每列最多 6 顆，
  **最後一列不足 6 顆時置中**（flex row + justify-content:center，
  列間距與顆距一致）。attempts=10 時即上排 0–5、下排 6–10 置中。
  拿掉現在的 grid-column span 補位邏輯（圓鈕下已無意義）。
- 修改輪次 sheet 用同一個 makesGridHtml，自動一致。
- 實投球數 stepper（1–20）同邏輯：每列 5–6 顆、末列置中，圓鈕等距。
- 觸控目標維持 ≥44px；深色模式檢查。

## 2. 熱區命中率字貫穿圓點（[court.js](../js/court.js)＋app.css）

使用者指定：% 字置中在圓點上，**允許左右突破圓的邊緣**（貫穿感）。

- `.spot-heat-pct`：26px → **34px**（手機實際約 15.5px），字重 800 不變，
  加 `paint-order: stroke` 深色細描邊（stroke 用球點底色的深階或
  rgba(0,0,0,0.35)、3px）讓超出圓的部分壓在球場線上也讀得清。
- 移除 100% 的 `--tight` 縮級機制——字可以超出圓了，不需要縮。
- 點半徑可回調至 **26**（字會突破圓，點不用撐那麼大），
  無資料點維持 r8；點擊資訊列機制不動。
- 相鄰點文字防撞檢查：ft(290)/mid_top(350) 縱向相鄰、「100%」約 82px 寬，
  橫向貫穿不影響縱向間距；左右相鄰最近的是 mid_lc(150,100)–paint(375,145)
  等，橫距 ≥185px，34px 字寬 ~82px 不會互撞（實作後仍要實測確認）。
- 分享卡迷你球場（sharecard.js drawMiniCourt）比照：% 字 18 → **24**
  （乘 scale 後約 29px），允許超出點，加同款深色描邊或陰影。

## 3. 分享卡佈局修正（[sharecard.js](../js/sharecard.js)）

### 3a. 修垂直重疊（本輪最重要）

- 主數字區塊結束 y 明確定義為「57% 大字與投中小字的 baseline + 44 以上」
  再開始球種列——投中行與球種列首行 baseline 差 **≥56px**。
- 佈局改用**顯式游標推進**：每畫完一個區塊，y += 該區塊實際高度＋固定
  gap（不要再用魔術數字疊加），並在檔內註明每段的高度預算表。
- 高度不夠就把迷你球場縮回 78–80% 卡寬——**寧可球場小一點，不准重疊**。
  最壞情境（4 球種＋2 徽章＋照片模式）必須全部有空隙。

### 3b. 球種列收緊

- 三欄錨點改**緊湊固定寬**，不撐滿欄：label 靠左（欄 x）、
  count 右對齊在「欄 x + 約 230」、pct 右對齊在「欄 x + 約 360」
  （實作時以 4 字元 label「深 3」與最寬 count「50/90」實測微調，
  兩欄位間至少留 24px），一列讀起來是連貫的一句。
- 兩欄 grid 維持，欄距固定（如 60px），整組**靠左**與其他區塊同 marginX，
  右側自然留白（不要為了填滿而拉開）。
- 對齊要求不變：跨列的 count 右緣、pct 右緣各自切齊。
