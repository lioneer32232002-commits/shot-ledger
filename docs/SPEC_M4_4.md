# SPEC M4.4 — 分享卡資訊區重排＋sheet 按鈕層級＋卡上熱區字放大（2026-07-14 第五輪回饋）

> 分工：Fable 寫 spec → Sonnet 實作 → Fable 驗收（含垂直帶掃描）。不 commit、不 push。

## 0. 使用者回饋原文重點

- 球種橫排時「罰球」label 落在卡片正中，視線被帶往下方球場點（不相干）。
- 球種要**上下排列**（每種一行），但要對齊。
- 分享 sheet「長型按鈕太多」。
- 卡上熱區 % 還是太小；App 統計頁「全期熱區」的相對大小是理想值。

## 1. 分享卡：球種列改「右欄直排」與主數字並排（[sharecard.js](../js/sharecard.js)）

主數字區與球種列合併為**同一水平帶、左右兩欄**：

- **左欄**（x = marginX 起）：57% 超大字（150px 不變），**「69/120 投中」
  移到大字正下方**（小字、同左對齊），不再擠在大字右側。
- **右欄**（x 約 500 起，實作時依大字實測寬度＋40px 起算、但固定錨點
  不隨 % 位數浮動——用「100%」最寬情境算死）：球種列**每種一行直排**，
  最多 4 行，行距約 44；三欄位固定錨點跨行對齊（label 靠左、
  顆數右對齊、% 右對齊，錨點間距沿用 M4.3 的緊湊值，欄寬以
  「深 3 50/90 56%」實測微調）。
- 整帶高度 = max(左欄實高, 右欄行數×行距)，用既有游標推進機制，
  高度預算表註解同步更新。
- 球種只有 1–2 種時右欄自然變短，不置中、維持頂對齊（與大字頂線齊）。
- 視線動線驗收：資訊全部集中在上帶，球場帶乾淨無夾雜文字。

## 2. 分享 sheet 按鈕層級重整（sharecard.js openShareSheet＋app.css）

現況五顆長鈕疊疊樂（照片、移除照片、分享、下載、關閉），主次不分。改為：

- **主鈕只有一顆**：「分享」primary（膠囊＋ring 照舊）。
  裝置不支援分享 API 時，「下載 PNG」升級成唯一主鈕。
- 「用自己的照片當背景」與「移除照片」：合併成**一列兩顆半寬**的
  次要鈕（secondary 描邊膠囊＋ghost）；未選照片時「移除照片」隱藏、
  照片鈕單顆置中半寬（不拉滿）。
- 「下載 PNG」（有分享 API 時）與「關閉」：**一列兩顆半寬**，
  下載用 secondary、關閉用 ghost 文字鈕。
- 目標：全寬長鈕最多一顆，其餘半寬並排，視覺重量分層。
- 觸控目標 ≥44px；深色模式檢查。

## 3. 卡上熱區 % 放大到與 App 同相對比例（sharecard.js drawMiniCourt）

換算：App 全期熱區字 34（SVG 750 座標）＝球場寬的 4.5%；
卡上目前 24＝3.2%，偏小的根源。

- drawMiniCourt 的 % 字 24 → **34**（SVG 座標，乘 scale 後約 41px），
  維持貫穿圓點，點 r 26 不變。
- **描邊全部移除**（使用者指定嘗試無描邊版）：
  - 卡上：drawMiniCourt 的 % 不再 strokeText，只 fillText 純白字；
  - App 球場：app.css `.spot-heat-pct` 的 stroke／paint-order 描邊一併拿掉。
  - 若無描邊在球場線上可讀性尚可就定案；驗收時記錄實際觀感。
- 相鄰點防撞：ft(290)/mid_top(350) 縱距 60×scale≈72px，字高 41px 貫穿
  橫向為主，縱向不撞；實作後仍以假資料實畫確認。
- 左右底角（cx 45/705）的「100%」貫穿後以 viewBox 邊界檢查，
  超出就把該兩點的文字錨點往內收（textAlign 改 left/right 於點緣），
  其他點不動。

## 驗收必檢（Fable）

1. 垂直帶掃描（2/4 球種 × 紙感/照片 4 張，相鄰帶間 ≥16px）。
2. 右欄跨行對齊掃描（顆數右緣、% 右緣）。
3. 球場帶內無資訊文字夾雜（動線檢查）。
4. sheet 按鈕：全寬鈕數量 ≤1、半寬列排版、無分享 API 情境（桌面）檢查。
5. 卡上熱區 % 相對比例 ≈ App 全期熱區（4.5% 球場寬）。
